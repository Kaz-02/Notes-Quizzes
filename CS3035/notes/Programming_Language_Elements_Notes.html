01 Token (Lexical Structure) Lexical

The smallest meaningful unit in a program — the "words" and "punctuation" of a programming language. A lexer/tokenizer breaks raw source text into tokens.

Keyword
int, for
Variable / Identifier
sum, i
Literal
0, 1, 10
Operator
=, +, <=, ++
Separator
; , ( ) { }
// Every piece below is a separate token:
int sum = 0;
02 Expression Syntax

A syntactic construct that evaluates to a single value. Expressions produce a result but do not (on their own) cause actions to happen.

  • Can be a literal: 0, 1, 10
  • Can be a variable: sum, i
  • Can be a compound expression: sum + i, i <= 10
  • Prefix increment ++i is also an expression (it evaluates to the new value of i)
From the example
sum + i evaluates to a number  ·  i <= 10 evaluates to a boolean
03 Statement Semantic

A single, complete instruction that tells the computer to perform a specific action (in an imperative language). Statements execute, they don't produce a value.

  • Statements change program state — these changes are called side effects
  • An assignment like sum = sum + i; is a statement
  • A declaration like int sum = 0; is a statement
int sum = 0;      ← declaration statement
sum = sum + i;  ← assignment statement (side effect: mutates sum)
04 Compound Statement Syntax

A statement composed of other statements and expressions. Allows multiple instructions to be grouped into one logical unit.

for(int i = 1 ; i <= 10 ; ++i) {  ← compound statement
    sum = sum + i;                     ← inner statement
}
  • The for loop header contains expressions: init, condition, update
  • The body { sum = sum+i; } is a block — another form of compound statement
05 Expression Statement Semantic

An expression that is also used as a statement — it both evaluates to a value AND has a side effect.

int i = 100;
int j = ++i;  ← ++i used as expression (j = 101)
++i;         ← ++i used as statement (value discarded, i = 102)
Expression Evaluates to Side Effect
i + 1 value (i+1) none
++i value (i+1) mutates i
++i; (as stmt) discarded mutates i
06 Function Structure

Encapsulates a computation behind a defined interface — you call it with inputs, it returns an output, hiding the implementation details.

def sum_to_n(n):
    total = 0
    for i in range(1, n + 1):
        total += i
    return total
  • Interface: function name + parameters (n) + return type
  • Body: the implementation — a compound statement
  • Callers use sum_to_n(10) without knowing how it works
  • This is the same logic as the running Java example, abstracted into a reusable function
07 Class & Object Structure

Bundles data (fields/attributes) and operations (methods) together into a single unit — the foundation of Object-Oriented Programming.

public class Car {
    String brand;         ← field (attribute)
    String model;         ← field (attribute)
    int    currentSpeed;   ← field (attribute)

    public void accelerate(int increment) { ← method
        currentSpeed += increment;
    }
}
Concept What it is Example
Class Blueprint / template Car
Object Instance of a class new Car()
Field Data stored in the object brand, currentSpeed
Method Operation on the object accelerate()
08 Module / Package / Library Structure

A higher-level organizational unit that groups related code. Enables code reuse across multiple programs and enforces separation of concerns.

  • Import — client code uses functionality from a module/library
  • Export — the module exposes selected data and operations to the outside world
# Python examples
import math               ← import a standard library module
from collections import Counter  ← import specific export

// Java
import java.util.ArrayList;  ← import from package
Key idea
The interface (what's exported) is public. The implementation (how it works inside) is hidden. This is called encapsulation.
