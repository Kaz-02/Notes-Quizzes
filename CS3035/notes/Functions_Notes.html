01

Parameters & Arguments

JS

Parameters are the variables listed in the function's definition. Arguments are the actual values supplied when the function is called.

function foo(x, y) {        // x and y are PARAMETERS
  return x + y;
}
foo(1, 2);                   // 1 and 2 are ARGUMENTS
02

Default Parameter Values

JSPy

A parameter can have a default value used when no argument is provided for it. Also known as a default argument.

function foo(x, y = 2, z = 3) {
  return x + y + z;
}
console.log(foo(4, 5));   // → ?
Answer
Output is 12. x = 4, y = 5 (overrides default of 2), z = 3 (uses default). So 4 + 5 + 3 = 12.
  • Default parameters should be listed after non-default parameters — otherwise callers can't skip them positionally.
  • In Python, defaults are evaluated once at function definition time.
  • In most other languages (JS, C++, etc.), defaults are evaluated at each function call.
03

Positional & Keyword Arguments

Py
foo(4, 5)            # positional — order matters
foo(y=5, x=1)       # keyword — order doesn't matter
  • Keyword arguments improve readability and safety, especially in functions with many parameters.
  • They pair well with default values — you can skip parameters you don't need to override.
04

Variadic Functions

JSPy

A variadic function accepts a variable number of arguments. Only one variadic parameter is permitted, and it must appear at the end of the parameter list.

Python — *args → tuple
def foo(x, *y):
  return [x, y]

foo(1, 2, 3)
# → [1, (2, 3)]
JavaScript — rest → array
function foo(x, ...y) {
  return [x, y];
}
foo(1, 2, 3);
// → [1, [2, 3]]
Terminology
JavaScript calls this a Rest Parameter (the ...spread syntax). Python uses *args. Both collect remaining arguments into a sequence.
05

In / Out Parameters

C#
KeywordDirectionMeaning
inInput onlyCaller's value is read inside the function; caller's variable is never modified.
outOutput onlyFunction must assign before returning; initial value of caller's variable is ignored.
ref / inoutInput & OutputCaller's variable is both read and written. Changes are reflected at the call site.
 
// Multiple return values via out
void Calc(in int a, in int b, out int sum, out int prod) {
  sum  = a + b;
  prod = a * b;
}

// Modifying caller's variable via ref
void Swap(ref int a, ref int b) {
  var temp = a;
  a = b;
  b = temp;
}

Swift uses the keyword inout for the same bidirectional purpose.

Parameter Passing Strategies

concept
StrategyMechanismLanguage / Example
Call by valueA copy of the argument is passed. Changes inside the function don't affect the caller.Most modern languages
Call by referenceThe function receives a reference to the caller's variable. Changes affect the caller.C/C++ pointers, ref / inout
Call by resultValue copied out to caller only on return; initial value not used inside.out parameter (C#)
Call by value-result"Copy-in / copy-out" — value copied in at call, result copied out on return.Ada
Call by nameThe argument expression is substituted textually and re-evaluated every time the parameter is used.ALGOL 60
Classic gotcha — reference vs. value-result with aliasing
foo(x, y) { x = x+1; y = y+2 }
z = 5;
foo(z, z);

// Call by reference  → both x and y alias z.
//   x = x+1  → z becomes 6
//   y = y+2  → z becomes 8 (y is still z)
//   Result: z = 8

// Call by value-result → copies in: x=5, y=5.
//   x = x+1 → x becomes 6
//   y = y+2 → y becomes 7
//   Copies out (last write wins): z = 7
//   Result: z = 7
Call by name — argument expression is re-evaluated each use
foo(a, b) { print(b); print(b); }
int x = -1, y = 3;
foo(++x, ++y);

// b is substituted with the expression ++y
// First  print(b) → ++y → y = 4, prints 4
// Second print(b) → ++y → y = 5, prints 5
// Final state: x = 0, y = 5

Recursion & Activation Records

JS

Recursion is when a function calls itself. Each call creates a new activation record (frame) on the call stack, storing that call's arguments and local variables.

function factorial(n) {
  if (n == 0) return 1;
  return n * factorial(n - 1);
}

Call stack for factorial(2) — grows as calls are made, unwinds as they return:

n = 0  ← top of stack returns 1
↕
n = 1 returns 1 × 1 = 1
↕
n = 2 returns 2 × 1 = 2
↕
Global frame bottom of stack
08

Tail Calls & Tail-Call Optimization

JS

A function call is a tail call if the caller immediately returns its result — with no additional computation after the call returns. Tail-call optimization (TCO) lets the runtime reuse the current frame instead of pushing a new one, keeping stack usage constant O(1).

Regular recursion — O(n) stack
function factorial(n) {
  if (n == 0) return 1;
  return n * factorial(n-1);
  // must multiply AFTER
  // recursive call returns
}
Tail recursion — O(1) stack
function factorial(n, acc=1) {
  if (n == 0) return acc;
  return factorial(n-1, n*acc);
  // result passed forward;
  // nothing to do on return
}
  • In the tail-recursive version, the accumulated result is carried as an argument (acc) — no pending multiplication on the way back.
  • TCO is mandatory in functional languages (Scheme, Haskell) and optional in others (some JS engines support it).

Functors & Operator Overloading

C++

A functor is an object that can be called like a function. The trick is overloading the function-call operator operator(). Unlike a plain function, a functor can hold and mutate state across calls.

class Counter {
public:
  int count = 0;

  int operator()() {
    return count++;      // state persists between calls
  }
};

Counter counter;
counter();   // returns 0 → count is now 1
counter();   // returns 1 → count is now 2

Operator overloading (used to enable functors) also lets you define arithmetic on custom types:

class Point {
public:
  double x, y;
  Point operator+(Point other) {
    return Point(x + other.x, y + other.y);
  }
};

Point p = p1 + p2;   // calls operator+
  • Functors are the foundation of C++ STL algorithms — they're passed as predicates and comparators.
  • In modern C++, lambdas are often preferred over explicit functor classes, but the underlying mechanism is the same.
