01 · Overview

About Python

Python is currently #1 on the TIOBE Index and #4 in the 2025 Stack Overflow Developer Survey. It is considered the primary language for AI and machine learning.

  • Has all the features of a modern programming language
  • Easy to use — even for non-programmers
  • An interpreted language (though technically hybrid — see below)

02 · Execution Model

Interpreted vs. Compiled Languages

🐍 Interpreted (Python, JS)

  • Translates source → binary one instruction at a time
  • Interpreter runs alongside the program
  • Faster to develop; runs slower

⚙️ Compiled (C, Java)

  • Translates the whole program to binary first
  • No interpreter needed at runtime
  • Slower to develop; runs faster
💡 Python and Java are actually hybrid — they compile source code to an intermediate bytecode format before execution.

03 · Getting Started

Hello World

The classic first program. It originated in The C Programming Language by Kernighan & Ritchie.

Why write it? It verifies your dev environment works, shows you basic output syntax, and gives a feel for the language.

print("Hello, World!")
public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}

Notice how much more concise Python is — no class wrapper, no type declarations, no semicolons.

07 · Control Flow

Statements

Python uses indentation (4 spaces recommended) to define blocks — no curly braces or semicolons.

if · elif · else

num = -5
if num > 0:
    print("Positive")
elif num == 0:
    print("Zero")
else:
    print("Negative")

match (Python 3.10+)

Like switch in Java/C but no break needed. _ is the default case.

day = 3
match day:
    case 1: print("Monday")
    case 2: print("Tuesday")
    case 3: print("Wednesday")
    case _: print("Invalid day")

for loop

Iterates over elements of any sequence or collection. There is no traditional for(int i=0; i<n; i++) — use range() instead.

fruits = ["apple", "banana", "cherry"]

for fruit in fruits:
    print(fruit)

for i in range(3):
    print(fruits[i])

while loop

Note: there is no do-while loop in Python.

count = 5
while count > 0:
    print(count)
    count -= 1
print("Liftoff!")

08 · Functions

Defining & Calling Functions

def square(x):
    return x * x

def greet(name, message):
    print(f"{message}, {name}!")

result = square(10)   # positional args
greet(message="Hi", name="Steve")  # keyword args
💡 If a function doesn't explicitly return a value, it implicitly returns None.

Default Parameter Values

Parameters with defaults must come after those without defaults.

def greet(name, message="Hi"):
    print(f"{message}, {name}!")

greet("Steve")           # → "Hi, Steve!"
greet("John", "Hello")   # → "Hello, John!"
⚠️ Never use a mutable object (like a list) as a default value. Default values are evaluated once at function definition — not on each call. This causes bugs where state persists across calls. 
def add_to_list(value, my_list=[]):  # ❌ BAD
    my_list.append(value)
    return my_list

print(add_to_list(1))  # [1]
print(add_to_list(2))  # [1, 2]  ← unexpected!
print(add_to_list(3))  # [1, 2, 3] ← still accumulating!

Docstrings

Used to document functions, classes, and modules. Must be the first statement. It's a string, not a comment.

def greet(name, message):
    """Display a simple greeting."""
    print(f"{message}, {name}!")

09 · OOP

Classes & Objects

class Dog:
    """A simple attempt to model a dog."""

    def __init__(self, name, age):   # constructor
        self.name = name            # instance variable
        self.age  = age

    def sit(self):                  # instance method
        print(f"{self.name} is now sitting.")

monty = Dog("Monty", 2)  # instantiation
monty.sit()
  • Objects of a class are called instances
  • Functions inside a class are called methods
  • __init__() is the constructor — runs when an object is created
  • self (the current object) must be the first parameter of every instance method
  • Instance variables are defined inside the constructor

10 · Modules & Packages

Organizing Code

Every Python file is a module. A folder of modules is a package (usually marked with an __init__.py).

foo.py (module)

def add(a, b):
    return a + b

PI = 3.14159

bar.py (importer)

import foo

print(foo.add(2, 3))
print(foo.PI)

Import styles

Import statementUsage
import foofoo.add(2,3)
import foo as ff.add(2,3)
from foo import addadd(2,3)
from foo import *add(2,3)
from foo import add as aa(2,3)

Package structure

my_app/
├── main.py
└── analytics/         # ← package
    ├── __init__.py
    ├── calculator.py
    └── formatter.py
# From analytics/calculator.py
def add(a, b): return a + b

# In analytics/__init__.py — "raise" to package level
from .calculator import add

# In main.py — clean import
from analytics import add
add(2, 3)
