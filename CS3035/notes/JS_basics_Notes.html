Types, Literals & Variables

Primitive Types (immutable values)

Boolean — true / false

Number — integers & floats (one type)

String — text

Null — intentional absence of value

Undefined — declared but not assigned

Symbol — unique identifiers (advanced)

Non-Primitive

Object — everything else: plain objects, arrays, functions, dates…


Use typeof x to check type at runtime.

Literals
// Boolean
true   false

// Number (one type for int & float)
123   4.56   -7

// String — single or double quotes
"hello"   'world'

// Null / Undefined
null   undefined

// Template Literal (backtick!) — embeds expressions with ${}
let a = 10, b = 20;
console.log( `${a}+${b} is ${a+b}` );  // "10+20 is 30"

// Object Literal
let car = { make: "Honda", model: "Civic", year: 2001 };
Template literals use backtick (`), not a single quote. Expressions inside ${ } are evaluated at runtime.
Variables & Constants
let x;           // declare (undefined)
x = 10;          // now a Number
x = 'abc';       // now a String — JS is dynamically typed!

const y = 20;   // constant — cannot be reassigned
Avoid var in modern JS — it has function scope and hoisting quirks. Prefer let and const.

03

Operators & Expressions

Equality: == vs ===
"5" == 5 true Loose equality — coerces types before comparing
"5" === 5 false Strict equality — same type AND same value required
5 === 5 true Same type, same value ✔
null == undefined true Special case — only pair that's loosely equal across types
Always prefer === and !== in modern JS. Loose equality (==) can cause subtle bugs.
Truthy & Falsy Values

JS has weak typing — non-boolean values are coerced to boolean in conditions.

FALSY — treated as false

false 0 "" null undefined NaN

TRUTHY — everything else, including:

"0" [] {} 42 "false"

Loops — JS vs Java

forClassic index loopSame as Java
while / do…whileCondition-basedSame as Java

Loop Examples
const arr = ['a', 'b', 'c'];

// for...of — iterate VALUES
for (const val of arr) console.log(val);  // a, b, c

// for...in — iterate KEYS (indices for arrays)
for (const key in arr) console.log(key);  // 0, 1, 2

// forEach — callback style
arr.forEach(val => console.log(val));       // a, b, c
The switch statement uses strict equality (===) for case matching, unlike C/Java which use value equality. Semicolons are optional but recommended.
05

Functions — First-Class Citizens

Functions Can Be Used Anywhere Values Can
Assigned to variables

A function can be stored in a variable, just like a number or string.

Passed as arguments

Pass a function to another function — this is what enables callbacks.

Returned from functions

A function can produce and return another function (closures, factories).

Object properties & array elements

When a function is stored in an object, it becomes a method.

Function Declaration vs Function Expression
// 1. Function declaration — hoisted, can be called before definition
function foo() {
  console.log("foo");
}

// 2. Function literal (expression) — NOT hoisted
let bar = function() {
  console.log("bar");
};

// 3. Passing a named function as argument
setTimeout(bar, 5000);

// 4. Passing an anonymous function literal as argument
setTimeout(function() {
  console.log("foobar");
}, 5000);
Arrow Functions (a.k.a. Lambdas)

Concise syntax for function literals. Three equivalent forms:

function(a) { return a*2 }
↓ remove "function", add =>
(a) => { return a*2 }
↓ single param → drop (), single expression → drop return & {}
a => a*2
⚠️ Arrow functions have no own this — they inherit this from outer scope. Never use them as object methods.
06

Objects, Methods & Accessor Properties

Object Literal

A collection of key : value pairs called data properties. Keys can be valid identifiers or strings/numbers.

let car = {
  make:    'Honda',        // identifier key
  model:   "Civic",
  "Year":  2001,           // string key (quoted)
  owner:   { name: "Chengyu" }  // nested object
};
Accessing Properties — Two Ways
// Dot notation — when key is a valid identifier
console.log(car.make);        // "Honda"

// Bracket notation — any key, or computed from a variable
console.log(car["make"]);     // "Honda"

let field = "model";
console.log(car[field]);      // "Civic" — computed property!

// Properties can be added dynamically
car.color = "red";
car[0] = "something";
JSON — Quick Comparison to Object Literal
JS Object Literal

Keys can be bare identifiers. Strings can use single or double quotes. Can include functions.

JSON

Keys must be double-quoted strings. Values must be double-quoted. No functions, no comments. Used for data exchange.

Method & this

A method is a function stored as an object property. Inside a method, this refers to the calling object.

let john = {
  firstName: "John",
  lastName:  "Doe",
  greeting: function() {
    console.log(`Hi, ${this.firstName}!`);  // this → john
  }
};

john.greeting();   // "Hi, John!"
⚠️ Do NOT use arrow functions for methods. Arrow functions capture this from the surrounding scope, not the calling object.
Accessor Property (Getter / Setter)

Accessed like a data property but backed by functions — great for computed or validated values.

circle-accessor.js
let circle = {
  radius: 5,

  get area() {                        // called when reading circle.area
    return Math.PI * this.radius ** 2;
  },

  set area(value) {                   // called when writing circle.area = x
    this.radius = Math.sqrt(value / Math.PI);
  }
};

console.log(circle.area);  // reads via getter  → ~78.54
circle.area = 100;         // writes via setter → updates radius
Accessor properties look like data properties to the caller — no parentheses needed. This is similar to Python's @property decorator.
07

Arrays

Array Basics

The most-used collection type in JS. More like Python list or Java ArrayList — dynamic size, mixed types.

let a = ["x", "y", "z"];

console.log(a[0]);       // "x"
console.log(a.length);   // 3

a.push("w");              // ["x","y","z","w"]
a.pop();                  // removes "w"
a.map(v => v + "!");     // ["x!","y!","z!"] (new array)
a.filter(v => v !== "y"); // ["x","z"] (new array)
Arrays Are Special Objects

Under the hood, array indices are just numeric string keys on an object. This has some surprising consequences:

a = ["x", "y", "z"];
// Internally stored as: { 0: "x", 1: "y", 2: "z" }

a.b = "hello";     // adds a non-index property (allowed but unusual!)

a[100] = 10;       // creates a "hole" — indices 3–99 are undefined
console.log(a.length);  // 101 — length = highest index + 1
Avoid creating sparse arrays with holes — many array methods skip empty slots inconsistently.
