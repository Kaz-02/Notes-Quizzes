Abstract Data Type

Study notes compiled from lecture slides

01

What is an Abstract Data Type (ADT)?

A data type defined by the data it stores and the operations that can be performed on it — without specifying how it is implemented.

The key idea is separation of interface from implementation. You define what a type can do, not how it does it.

What an ADT specifies

  • What data is stored
  • What operations are available
  • What each operation does (behavior)

What an ADT does NOT specify

  • How data is stored internally
  • Which data structure is used
  • How operations are implemented
02

ADT Example: Stack

A Stack is a classic ADT. Think of a stack of plates — you can only add or remove from the top. This is known as LIFO (Last In, First Out).

Stack state

Operations (interface)
push(x)Add x to the top
pop()Remove & return top element
peek()View top element (no removal)
size()Return number of elements
These four operations are the interface of the Stack ADT. Anyone using a Stack only needs to know these — not how they're implemented internally.
03

ADT vs. Data Structure

ADT (Abstract)

Defines the concept — what data and operations exist.

No implementation details.

Think: a Stack that supports push, pop, peek, size

Data Structure (Concrete)

Defines the implementation — how data is organized and operations work.

Examples: Array, Linked List, Binary Tree

The same ADT can be implemented by multiple different data structures. The choice affects performance, not behavior:

ADTPossible Data Structure Implementations
StackArray (with top pointer), Singly Linked List
QueueArray (circular), Doubly Linked List
SetHash Table, Balanced BST (e.g., Red-Black Tree)
Map / DictionaryHash Table, B-Tree, Skip List

Example: Implementing Stack with an Array

One concrete way to implement the Stack ADT is using an array with a top index pointer.

top = 2  |  size = 3  |  capacity = 6

  • top pointer — tracks the index of the current top element
  • push(x) increments top, places x at array[top]
  • pop() returns array[top], then decrements top
  • When array is full — allocate a new array, usually double the size, and copy elements over (amortized O(1))

Java — array-based stack (simplified)

class ArrayStack {
    private int[] data;
    private int top = -1;

    ArrayStack(int capacity) { data = new int[capacity]; }

    void push(int x) {
        if (top == data.length - 1) resize(); // double the array
        data[++top] = x;
    }

    int pop()  { return data[top--]; }
    int peek() { return data[top]; }
    int size() { return top + 1; }
}
The user of this Stack sees only push(), pop(), peek(), and size(). The array, the top pointer, and the resize logic are all hidden — this is encapsulation at work.

Application Programming Interface (API)

  • Defines the programming interface at different levels: a data type, a module, a library, or an entire web service
  • Hides implementation — callers don't need to know how it works
  • Acts as a contract between software components — as long as the API is respected, either side can change internally

Data typeJava Stack interfacepush, pop, peek, size
LibraryJava Collections APIList, Map, Set and their operations

07

OOP is Ideal for Implementing ADTs

Object-oriented languages are well-suited for ADTs because they provide the exact mechanisms needed:

📦
Bundling

Classes bundle data (fields) and operations (methods) into a single unit.

🔒
Encapsulation

Access modifiers (private, protected) hide internal implementation details.

🔄
Polymorphism

Multiple implementations (e.g., ArrayList vs LinkedList) can satisfy the same interface.

Java — ADT defined as interface, implemented as class

// ADT → Java Interface (the "what")
interface Stack<T> {
    void    push(T x);
    T       pop()  throws EmptyStackException;
    T       peek();
    int     size();
}

// Data Structure → Class (the "how")
class ArrayStack<T> implements Stack<T> {
    private T[] data;       // hidden from users ✅
    private int top = -1;   // hidden from users ✅

    public void push(T x)  { /* … */ }
    public T    pop()       { /* … */ }
    public T    peek()      { /* … */ }
    public int  size()      { /* … */ }
}

// Client code — programs to the interface, not the implementation
Stack<Integer> s = new ArrayStack<>();
s.push(10);
s.push(20);
System.out.println(s.pop()); // → 20
