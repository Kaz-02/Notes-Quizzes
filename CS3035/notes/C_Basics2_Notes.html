About C

Popularity
#2 TIOBE · #11 SO 2025

Consistently one of the most used languages despite its age (invented ~1972).

Strength
Low-level & fast

Direct memory access and minimal runtime overhead make it ideal for systems programming.

Legacy
Basis for everything

Linux, Windows, macOS kernels, the Python and Java runtimes, and most infrastructure libraries are written in C.

Risk
Security vulnerabilities

Manual memory management enables devastating bugs: Morris Worm (1988), Heartbleed (2014), and many more.

The C Standards

C has evolved through several standardized revisions. Most modern code targets C11 or C17:

StandardYearPurpose
C89 / C901989First official standardization of the language
C991999Modernization — inline functions, // comments, bool, variable-length arrays
C112011Concurrency support, improved safety, Unicode
C17 / C182017Bug fixes and clarifications — no new features
C232023Cleanup and language evolution
Tip: When creating a new project in an IDE (e.g. CLion), you'll be asked to choose a standard. C11 or C17 are safe and widely supported choices.
03

Hello World & printf()

hello.c 
#include <stdio.h>     // like import — gives us printf(), scanf(), etc.

int main() {
  printf("Hello, World!\n");
  return 0;              // exit code: 0 = success, non-0 = error
}
main()
Required entry point

Every C program must have exactly one main() function — this is where execution begins.

\n
Escape sequences

\n = newline, \t = tab, \\ = backslash, \" = quote. Unlike Python's print(), printf() does not add a newline automatically.

return 0
Exit code

Returning 0 signals success to the OS. Non-zero values signal errors.

printf() format specifiers:

char   name[] = "John";
int    age    = 20;
double gpa    = 3.1;
char   grade  = 'A';
unsigned int x = 42;

printf("Name:  %s\n",     name);   // %s  = string (char array)
printf("Age:   %d\n",     age);    // %d  = signed decimal int
printf("GPA:   %f\n",     gpa);    // %f  = float/double
printf("GPA:   %.3f\n",   gpa);    // %.3f = 3 decimal places → 3.100
printf("Grade: %c\n",     grade);  // %c  = single char
printf("Uint:  %u\n",     x);      // %u  = unsigned decimal

Static vs Dynamic Typing

C — Static Typing

int add(int x, int y) {
  int sum = x + y;
  return sum;
}
// Types checked at compile time

Python — Dynamic Typing

def add(x, y):
  sum = x + y
  return sum
# Types checked at runtime
FeatureStatic (C, Java)Dynamic (Python, JS)
Type checkingCompile timeRuntime
Error detectionEarlier — before runningLater — only when executed
VerbosityMore (explicit types)Less (types inferred)
FlexibilityLowerHigher
PerformanceUsually fasterUsually slower
ToolingStrong (autocomplete, refactoring)Weaker without type hints
05

Comments in C

Single-line
// comment

Added in C99. Everything after // on a line is ignored.

Multi-line
/* comment */

Available since C89. Spans any number of lines. Cannot be nested.

// Single-line comment

/* Multi-line comment
   spanning several lines */

int x = 10; // inline comment
⚠ C has no JavaDoc equivalent. Documentation tools like Doxygen use their own comment syntax (/** */), but this is a convention, not part of the C standard.

Types & Literals

C has no built-in String or bool types — strings are char arrays, and booleans are integers (0 = false, non-zero = true). C99 added <stdbool.h> for convenience.

TypeTypical SizeDescription
char1 byteInteger type — also used to store single characters
short2 bytesSmall integer
int4 bytesStandard integer (most common)
long4 or 8 bytesPlatform-dependent size
long long8 bytesGuaranteed 64-bit integer
float4 bytes32-bit floating point
double8 bytes64-bit floating point (preferred)
long double12 or 16 bytesExtended precision
void—No value (used for functions with no return, and void pointers)
Why so many numeric types? C targets embedded systems, OS kernels, and hardware where precise control over memory layout and integer width is essential.

Literal examples:

// Integer suffixes
int          a = 123;     // decimal
int          b = 0x10;    // hex  (16)
int          c = 010;     // octal (8)
int          d = 0b10;    // binary (2) — C23 / GCC extension
unsigned int e = 10u;     // u = unsigned
long         f = 10l;     // l = long
unsigned long g = 10ul;   // ul = unsigned long

// char is an integer too
char ch1 = 'a';           // character literal
char ch2 = 65;            // same as 'A' — ASCII value

// Floating point
double x = -4.56;
double y = 1.5e3;         // = 1500.0
float  z = 3.14f;         // f suffix = float
long double w = 3.14l;    // l suffix = long double
07

Arrays & Strings

Array declaration & initialization 
int a[] = {1, 2, 3, 4};         // size inferred = 4
int b[6] = {1, 2, 3};           // → {1, 2, 3, 0, 0, 0} rest zero-filled
int c[6];                       // uninitialized — local array has garbage values!

int[] d;  // ❌ Error — array needs a size or initializer
int  e[]; // ❌ Syntax error
⚠ Uninitialized local arrays contain garbage values — unlike global arrays which are zero-initialized. Always initialize explicitly.
Compound literal (anonymous array) 
// Pass an array inline without naming it — like Python's [1,2,3]
print_sum( (int[]){1, 2, 3}, 3 );

Strings in C — there is no String type. A string is just an array of char terminated by a null byte ('\0', i.e., byte value 0):

char s1[] = "hello";                       // 6 bytes: h e l l o \0
char s2[] = {'h','e','l','l','o','\0'};  // exactly the same
Key distinction: '0' is the character digit zero (ASCII 48). '\0' is the null terminator (ASCII 0, byte value 0). Mixing them up corrupts strings.
08

Identifiers & Naming Conventions

Identifiers may use letters, digits, and _ — but cannot start with a digit. C naming is more relaxed than Java (no enforced standard), but common conventions are:

snake_case my_variable, calc_sum() → Variables, functions (almost everything)
UPPER_CASE MAX_SIZE, PI → Constants & macros (#define)
PascalCase LinkedList, Point → Struct / union names (some styles)
_t suffix size_t, uint8_t → Type aliases (typedef)

Operators & Expressions

Arithmetic
+ - * / %

Standard math. Division on integers truncates: 7/2 = 3.

Assignment
= += -= …

Assignment is also an expression (has a value). Enables patterns like while(c = getchar()){…}.

Comparison
== != > >= …

Returns 1 (true) or 0 (false). No boolean type in old C.

Logical
&& || !

Any non-zero value is truthy — unlike Java which requires strict booleans.

Conditional
cond ? a : b

Ternary — compact inline if/else.

Increment
++ --

Pre vs post matters: ++x increments before using, x++ uses then increments.

Pointer-related operators (more detail in the pointers lecture):

Operator
& — address of

Gets the memory address of a variable. &x gives a pointer to x.

Operator
* — dereference

Follows a pointer to get its value. *ptr reads the value stored at the address in ptr.

Operator
-> member access

Access a struct member through a pointer: ptr->field is shorthand for (*ptr).field.

⚠ C truthy/falsy: In C, any non-zero integer is "truthy". This allows while(c = getchar()) — the assignment itself is the condition. This would be a bug in Java.

Statements & Control Flow

if-else

int num = -5;
if (num > 0) {
  printf("Positive");
} else if (num == 0) {
  printf("Zero");
} else {
  printf("Negative");
}
// {} optional for single-line blocks

switch

int day = 3;
switch (day) {
  case 6:
    printf("Saturday");
    break;      // ← required!
  case 7:
    printf("Sunday");
    break;
  default:
    printf("Weekday");
}
⚠ switch fallthrough: Without break, execution "falls through" into the next case. This is sometimes intentional, but is a common bug.

for loop

int arr[] = {10,20,30,40};
int n = sizeof(arr) / sizeof(arr[0]);

for (int i=0; i<n; i++) {
  printf("%d ", arr[i]);
}
// No for-each in C!
// sizeof evaluated at COMPILE time

while / do-while

int count = 5;
while (count > 0) {
  printf("%d\n", count);
  count--;
}

int sum=0, i=1;
do {
  sum += i++;
} while (i <= 5);
// do-while always runs at least once
No for-each loop in C. C has no built-in collection types or iterators, and array size is not stored at runtime. You must manually track the size (using sizeof at compile time, or passing it as a parameter).
11

Functions

Functions must be declared or defined before they are called. There are no default parameter values or keyword arguments in C.

C 
// Return type | name | parameters
int square(int x) {
  return x * x;
}

// void = no return value
void greet(char name[], char message[]) {
  printf("%s, %s!", name, message);
}

int main() {
  int result = square(5);         // → 25
  greet("Alice", "Good morning");
  return 0;
}
Arrays as parameters: When you pass an array to a function, C passes only a pointer to the first element — not a copy. The function does not know the array's length, so you must pass the size separately.

Struct & Union

Struct — like a class with only public fields and no methods. Each member has its own memory slot.

struct (verbose)

struct Point {
  double x;
  double y;
};

struct Point p1;
p1.x = 10.1;
p1.y = 11.2;

typedef (cleaner)

typedef struct {
  double x;
  double y;
} Point;

Point p1;   // no "struct" needed
p1.x = 10.1;
p1.y = 11.2;

Union — all members share the same memory location. Only one member is "active" at a time. Memory allocated = size of the largest member.

union definition

union Data {
  int   i;    // 4 bytes
  float f;    // 4 bytes
  char  c;    // 1 byte
};
// Total size = 4 bytes (max member)

union Data d;
d.i = 10;   // sets int
d.f = 11.2; // overwrites — d.i now garbage

Struct vs Union

// Struct: separate slots
struct { int a; float b; };
// Memory: [a a a a][b b b b] = 8 bytes

// Union: shared slot
union { int a; float b; };
// Memory: [a/b a/b a/b a/b] = 4 bytes

Real-world example: A network message that can carry different payload types:

enum DataType { TYPE_INT, TYPE_FLOAT, TYPE_STRING };

union Payload {
  int  i;
  float f;
  char str[20];
};

struct Message {
  enum DataType type;   // tells us which field is active
  union Payload data;   // only uses memory for the largest case
};

Libraries & Header Files

C splits a library into two parts: the implementation (.c files) and the API declaration (.h header file). Users #include the header and link against the compiled library.

math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int add(int x, int y);
int subtract(int x, int y);

#endif
math_utils.c
#include "math_utils.h"

int add(int x, int y) {
  return x + y;
}
int subtract(int x, int y) {
  return x - y;
}
app.c
#include <stdio.h>
#include "math_utils.h"

int main() {
  int s = add(3, 4);
  printf("Sum: %d\n", s);
}
Angle brackets vs quotes: #include <stdio.h> — system library (compiler looks in standard paths). #include "math_utils.h" — your own file (compiler looks in current directory first).

Compile & link pipeline:

app.c
+
math_utils.c
→
gcc compile
→
app.o + math_utils.o
→
gcc link
→
./app
Terminal 
gcc app.c math_utils.c -o app    # compile + link in one step

Header Guard — prevents the same header from being included more than once (causes duplicate declaration errors):

#ifndef MATH_UTILS_H    // if MATH_UTILS_H is not yet defined…
#define MATH_UTILS_H    // …define it (so future includes are skipped)

int add(int x, int y);
int subtract(int x, int y);

#endif                   // end of the guard
Modern alternative: Many compilers support #pragma once as a one-liner that does the same thing, though it's not officially in the C standard.
